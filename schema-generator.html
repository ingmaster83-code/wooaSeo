구조화 데이터 생성기 — JSON-LD 스키마 자동 생성

WebSite, Article, Product, FAQ, LocalBusiness 등 다양한 Schema.org JSON-LD 코드를 무료로 생성합니다. 복사해서 <head>에 바로 붙여넣으세요.

스키마 유형 선택

Article 정보 입력
Product 정보 입력
FAQ 항목 입력
LocalBusiness 정보 입력
브레드크럼 항목 입력
생성된 JSON-LD 코드 

    
<head> 태그 안에 붙여넣으세요.

  


  

    
💡 구조화 데이터 활용 팁

    
    
      
  • JSON-LD 방식이 Google 공식 권장 방식입니다. HTML 본문을 수정하지 않고 <head>에 추가만 하면 됩니다.
    • 
      
  • WebSite 스키마의 검색 URL 패턴을 추가하면 구글 검색창에 사이트 내 검색 박스가 표시될 수 있습니다.
    • 
      
  • Article 스키마는 발행일/수정일이 있을수록 뉴스/블로그 리치 스니펫 적용 가능성이 높아집니다.
    • 
      
  • Product 스키마의 평점·가격은 검색결과에 별점·가격 정보로 표시될 수 있습니다.
    • 
      
  • Google 리치 결과 테스트로 유효성을 반드시 확인하세요.
    • 
    

  


  

    
자주 묻는 질문

    

      
구조화 데이터를 추가하면 반드시 리치 스니펫이 표시되나요? ▾

      
아닙니다. 구조화 데이터는 리치 스니펫의 자격 요건을 갖추는 것이며, 실제 표시 여부는 Google의 재량입니다. 콘텐츠 품질, 페이지 권위도 등이 복합적으로 작용합니다.

    

    

      
여러 스키마를 한 페이지에 함께 사용할 수 있나요? ▾

      
네, 가능합니다. <script type="application/ld+json"> 블록을 여러 개 추가하거나, @graph 배열로 묶어서 사용할 수 있습니다.

    

    

      
FAQPage 스키마는 어디에 사용하나요? ▾

      
블로그 글, 상품 페이지, 서비스 소개 페이지 등 FAQ 섹션이 있는 페이지에 사용합니다. Google 검색결과에서 아코디언 형태로 질문/답변이 펼쳐지는 리치 스니펫이 표시될 수 있습니다.

    

    

      
BreadcrumbList는 무조건 추가하는 게 좋은가요? ▾

      
디렉토리 구조가 있는 사이트라면 권장합니다. 검색결과 URL 아래에 홈 > 카테고리 > 페이지 형태의 경로가 표시되어 클릭률(CTR) 향상에 도움이 됩니다.
