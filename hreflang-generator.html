hreflang 태그 생성기 — 다국어 SEO 링크 자동 생성

언어·지역별 URL을 입력하면 <head> 삽입용 link 태그와 sitemap XML용 xhtml:link 코드를 동시에 생성합니다.

언어/지역별 URL 입력
link rel="alternate" 코드 

      
각 언어 페이지의 <head> 안에 모든 코드를 동일하게 삽입하세요.

    


    

      

        Sitemap XML xhtml:link 코드
        
      

      

      
각 <url> 블록 안에 삽입하세요. sitemap.xml 최상단에 xmlns:xhtml 네임스페이스를 추가해야 합니다.

    

  


  

    
💡 hreflang 사용 팁

    
    
      
  • hreflang 태그는 모든 언어 버전 페이지에 동일하게 삽입해야 합니다. 한 페이지에만 있으면 효과가 없습니다.
    • 
      
  • x-default는 어떤 언어도 일치하지 않을 때 보여줄 기본 페이지 URL입니다. 보통 영문 페이지나 언어 선택 페이지를 지정합니다.
    • 
      
  • 언어 코드(ko, en)만 또는 언어+지역(ko-KR, en-US)을 함께 사용할 수 있습니다. Google은 둘 다 인식합니다.
    • 
      
  • 같은 페이지에 자기 자신을 가리키는 hreflang도 반드시 포함해야 합니다.
    • 
      
  • Google Search Console에서 국제 타겟팅 섹션을 통해 hreflang 오류를 확인할 수 있습니다.
    • 
    

  


  

    
자주 묻는 질문

    

      
hreflang 태그가 없으면 어떻게 되나요? ▾

      
Google이 각 언어 버전을 중복 콘텐츠로 인식하거나, 잘못된 언어 페이지를 해당 국가 검색결과에 표시할 수 있습니다. 다국어 사이트라면 hreflang 구현이 필수입니다.

    

    

      
hreflang을 sitemap.xml에 추가하는 방법은 무엇인가요? ▾

      
sitemap.xml 최상단 urlset 태그에 xmlns:xhtml="http://www.w3.org/1999/xhtml"를 추가하고, 각 <url> 블록 안에 생성된 xhtml:link 코드를 삽입하면 됩니다. 이 도구의 "Sitemap XML용" 탭에서 코드를 복사하세요.

    

    

      
언어 코드와 지역 코드를 함께 써야 하나요? ▾

      
언어만(ko, en)으로도 충분한 경우가 많지만, 같은 언어권의 다른 지역을 구분할 때는 en-US, en-GB처럼 지역 코드를 추가하세요. 한국어는 대부분 ko 또는 ko-KR을 사용합니다.

    

    

      
hreflang 적용 후 효과가 나타나는 데 얼마나 걸리나요? ▾

      
Google이 모든 언어 버전 페이지를 재크롤링해야 하므로 보통 수 주가 소요됩니다. Google Search Console 국제 타겟팅 보고서에서 오류 여부를 먼저 확인하세요.
